# NewLife.Office 需求文档

## 1. 项目定位

NewLife.Office 是一个轻量级办公自动化库，聚焦 Excel（xlsx）文件的导入导出。  
**核心定位**：以**输出（导出）为主、输入（导入）为辅**，解决业务系统中最常见的报表导出、数据导入场景。  
**设计原则**：零外部依赖（仅依赖 NewLife.Core），直接操作 Open XML（ZIP + XML），保持轻量与高性能。  
**说明**：CSV 读写操作已由 NewLife.Core 中的 `CsvFile` 类提供支持，本项目不重复实现。

## 3. 需求规划

基于"以输出为主、输入为辅"的定位，结合业务系统中最常用的场景，将功能需求分为三个阶段。

### 3.1 第一阶段：核心增强（高优先级）

> 解决最常见的导出场景痛点，让导出的Excel"好看、能用"。

#### 3.1.1 单元格样式（写入）

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **字体设置** | 字体名称、大小、粗体、斜体、颜色、下划线 | 🔴 高 |
| **填充/背景色** | 单元格背景色，至少支持纯色填充 | 🔴 高 |
| **边框** | 四边边框样式（实线/虚线/无）与颜色 | 🔴 高 |
| **对齐方式** | 水平对齐（左/中/右）、垂直对齐（上/中/下）、自动换行 | 🔴 高 |
| **自定义数字格式** | 导出时指定格式（如`#,##0.00`、`yyyy-MM-dd`） | 🟡 中 |

典型场景：导出报表时表头加粗加背景色、金额右对齐、日期列指定格式。

#### 3.1.2 合并单元格（写入）

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **合并单元格** | 指定范围合并，写入标题行、分类汇总行 | 🔴 高 |
| **读取合并信息** | 导入时识别合并区域，正确分配值 | 🟡 中 |

典型场景：报表标题横跨多列，分组表头合并。

#### 3.1.3 行高设置（写入）

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **设置行高** | 手动指定行高（磅值） | 🟡 中 |
| **自动行高** | 根据内容自动估算行高（多行文本场景） | 🟢 低 |

#### 3.1.4 冻结窗格

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **冻结首行** | 数据量大时冻结表头，方便滚动查看 | 🔴 高 |
| **冻结首列** | 冻结左侧若干列 | 🟡 中 |
| **自定义冻结** | 指定冻结行+列 | 🟡 中 |

#### 3.1.5 自动筛选

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **设置自动筛选** | 为表头行启用自动筛选功能 | 🟡 中 |

典型场景：导出数据列表后，用户需要在 Excel 中快速筛选。

### 3.2 第二阶段：便捷 API（中优先级）

> 提供更高层的抽象，减少业务代码编写量。

#### 3.2.1 实体/对象映射

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **对象集合导出** | `IEnumerable<T>` → Excel，属性自动映射为列 | 🔴 高 |
| **Attribute 映射** | `[DisplayName]`/`[Description]` 控制列名、顺序、格式 | 🔴 高 |
| **对象集合导入** | Excel → `List<T>`，按列名/序号映射到属性 | 🟡 中 |
| **忽略属性** | 标记某些属性不参与导出/导入 | 🟡 中 |
| **自定义转换器** | 对特定列/属性指定转换逻辑 | 🟢 低 |

典型场景：
```csharp
// 导出
var list = db.Users.ToList();
writer.WriteObjects("用户", list);

// 导入
var users = reader.ReadObjects<UserInfo>();
```

#### 3.2.2 DataTable 支持

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **DataTable 导出** | DataTable → Excel 工作表 | 🟡 中 |
| **DataTable 导入** | Excel 工作表 → DataTable | 🟡 中 |

#### 3.2.3 模板填充

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **模板填充导出** | 基于 xlsx 模板，替换占位符或填充区域 | 🟡 中 |
| **列表区域填充** | 模板中标记数据区域，自动插入/扩展行 | 🟢 低 |

典型场景：固定格式的报表（如月报、对账单），预制模板，只填充数据部分。

#### 3.2.4 超链接

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **写入超链接** | URL 链接、邮箱链接、工作表内部链接 | 🟡 中 |
| **读取超链接** | 导入时提取单元格超链接地址 | 🟢 低 |

#### 3.2.5 数据验证

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **下拉列表** | 为单元格设置下拉选项，约束输入 | 🟡 中 |
| **数值范围** | 限制输入的数值范围 | 🟢 低 |
| **日期范围** | 限制输入的日期范围 | 🟢 低 |

典型场景：导出模板让用户填写，通过下拉列表约束"状态"列等枚举值。

### 3.3 第三阶段：高级功能（低优先级）

> 按需实现，提升竞争力。

#### 3.3.1 图片操作

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **插入图片** | 向单元格或指定位置插入图片（PNG/JPEG） | 🟡 中 |
| **调整图片大小** | 控制图片尺寸、位置 | 🟢 低 |
| **读取图片** | 导入时提取嵌入图片 | 🟢 低 |

#### 3.3.2 打印与页面设置

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **页面方向** | 横向/纵向 | 🟡 中 |
| **纸张大小** | A4、A3、Letter 等 | 🟡 中 |
| **页边距** | 上下左右边距 | 🟢 低 |
| **页眉页脚** | 自定义页眉页脚文本 | 🟢 低 |
| **打印区域** | 指定打印范围 | 🟢 低 |
| **打印标题行** | 每页重复打印表头 | 🟡 中 |

#### 3.3.3 性能优化

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **流式写入** | 写入时不全量缓存行 XML，降低大文件内存占用 | 🟡 中 |
| **异步写入** | 提供 async API | 🟢 低 |
| **大文件读取优化** | SAX 模式解析 XML，避免 XDocument 全量加载 | 🟡 中 |

#### 3.3.4 工作表保护

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **工作表保护** | 防止用户修改特定区域 | 🟢 低 |
| **工作簿密码** | 打开文件需要密码 | 🟢 低 |

#### 3.3.5 条件格式

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **数值条件** | 大于/小于/等于时高亮 | 🟢 低 |
| **数据条** | 值大小可视化 | 🟢 低 |
| **色阶** | 按值渐变颜色 | 🟢 低 |

## 5. 常用 API 示例（基于已实现功能）

### 5.1 单元格样式

```csharp
/// <summary>单元格样式</summary>
public class CellStyle
{
    /// <summary>字体名称</summary>
    public String? FontName { get; set; }

    /// <summary>字体大小（磅）</summary>
    public Double FontSize { get; set; }

    /// <summary>是否粗体</summary>
    public Boolean Bold { get; set; }

    /// <summary>是否斜体</summary>
    public Boolean Italic { get; set; }

    /// <summary>字体颜色（ARGB 如 "FF0000" 表示红色）</summary>
    public String? FontColor { get; set; }

    /// <summary>背景色（ARGB）</summary>
    public String? BackgroundColor { get; set; }

    /// <summary>水平对齐</summary>
    public HorizontalAlignment HAlign { get; set; }

    /// <summary>垂直对齐</summary>
    public VerticalAlignment VAlign { get; set; }

    /// <summary>是否自动换行</summary>
    public Boolean WrapText { get; set; }

    /// <summary>边框样式</summary>
    public CellBorderStyle Border { get; set; }

    /// <summary>自定义数字格式</summary>
    public String? NumberFormat { get; set; }
}
```

### 5.2 写入 API 扩展

```csharp
// 写入带样式的表头
writer.WriteHeader("Sheet1", headers, headerStyle);

// 写入带合并单元格的标题行
writer.MergeCell("Sheet1", "A1:F1");
writer.WriteRow("Sheet1", new Object?[] { "月度报表", null, null, null, null, null }, CellStyle.Title);

// 冻结首行
writer.FreezePane("Sheet1", 1);

// 设置自动筛选
writer.SetAutoFilter("Sheet1", "A1:F1");

// 设置行高
writer.SetRowHeight("Sheet1", 1, 30);
```

### 5.3 对象映射 API

```csharp
// 导出
var list = GetUsers();
writer.WriteObjects("用户列表", list);

// 导入
var users = reader.ReadObjects<UserInfo>("用户列表");
```

## 6. 不做的功能

以下功能超出项目定位，明确不纳入规划：

| 功能 | 原因 |
|------|------|
| xls（97-2003）格式支持 | 二进制格式复杂度高，收益低，xlsx 已成主流 |
| 公式计算引擎 | 复杂度极高，非导出场景核心需求 |
| 图表生成 | Open XML 图表规范极为复杂，建议使用 Excel 端生成 |
| 数据透视表 | 规范复杂度极高，应用场景有限 |
| VBA/宏 | 安全风险高，非数据导出需求 |
| OLE 对象 | 过于专业，几乎无场景 |
| 完整主题/样式表 | 部分支持常用样式即可，无需实现完整主题系统 |

## 7. 与竞品差异化优势

| 对比维度 | NewLife.Office | 竞品 |
|----------|---------------|------|
| **依赖** | 仅 NewLife.Core，无第三方依赖 | EPPlus/ClosedXML 依赖 Open XML SDK |
| **体积** | 极小（单文件级别） | EPPlus/NPOI 数 MB |
| **框架兼容** | net45 ~ net10 全覆盖 | 多数仅 netstandard2.0+ |
| **许可** | MIT 完全免费 | EPPlus v5+ 商业许可 |
| **学习成本** | API 极简，几行代码完成导出 | ClosedXML/NPOI API 较重 |
| **生态集成** | 与 NewLife.XCode 等深度集成 | 独立库，需自行胶水代码 |
| **长数字处理** | 内置身份证/长整型保护 | 需手动处理 |

## 1. 项目定位

NewLife.Office 是一个功能全面的 .NET 办公自动化库，覆盖 **Excel（xlsx/xls）、Word（docx/doc）、PPT（pptx/ppt）、PDF**、Markdown、RTF、ODS、Email（eml/msg）、iCalendar、vCard、EPUB、XPS 等主流文档格式，以及基于 OLE2/BIFF8 的老式二进制格式（.xls/.doc/.ppt）。  
**核心定位**：以 MIT 开源协议为社区提供主流办公文档的**全面操作能力**，覆盖创建、读取、编辑、模板填充、合并拆分等常见业务场景，让开发者无需依赖商业库即可完成绝大多数办公自动化任务。  
**设计原则**：零外部依赖（仅依赖 NewLife.Core），直接操作 Open XML（ZIP + XML）/ PDF / OLE2 文件结构，兼顾功能全面与高性能。  
**说明**：CSV 读写操作已由 NewLife.Core 中的 `CsvFile` 类提供支持，本项目不重复实现。

## 2. Excel 功能需求

### 2.1 文件读写

- 读取和写入 xlsx 格式（Open XML，直接操作 ZIP + XML）
- 读取和写入 xls 格式（OLE2/BIFF8，兼容 Excel 97-2003）
- 多工作表支持（按名称/索引访问）
- 共享字符串处理
- 流式读写（低内存消耗，适合大文件）

### 2.2 单元格与样式

- 单元格类型自动识别（文本、数值、日期、布尔、百分比）
- 长数字保留文本（防科学计数法）、身份证号/前导零保护
- 字体（名称/大小/粗体/斜体/下划线/颜色）、背景填充色、边框
- 对齐方式（水平/垂直/自动换行）
- 自定义数字格式（`#,##0.00`、`yyyy-MM-dd` 等）
- 条件格式（数值条件高亮、数据条、色阶）
- 富文本（单元格内多种格式混排）

### 2.3 布局与布局操作

- 列宽（自动估算、手动设置）、行高（手动/自动）
- 合并单元格（读取/写入）
- 冻结窗格（行、列、行+列）
- 自动筛选
- 隐藏行/列、分组折叠

### 2.4 内容元素与高级能力

- 公式（读写，不含计算引擎）
- 超链接（URL、邮箱、工作表内部跳转）
- 数据验证（下拉列表、数值范围、日期范围）
- 图片插入与读取（drawing/media）- **图表**（柱状/折线/饼图/面积/散点图，创建与读取）
- **数据透视表**（基本创建与读取）- 对象映射导入/导出（`IEnumerable<T>` ↔ Excel，支持 `[DisplayName]`/`[Description]`）
- 模板填充（基于 xlsx 模板的占位符替换）
- 导出为 CSV（通过 NewLife.Core CsvFile 支持）

### 2.5 高级数据结构（高保真差距补齐，M18-M19）

- **命名范围（Defined Names）**：`AddDefinedName(name, range)` 创建命名范围并写入 `workbook.xml`；`ReadDefinedNames()` 读取；按名称访问范围
- **结构化表格（Table）**：`AddTable(range, name, style)` 创建带样式的 `<table>` 元素及独立 `table*.xml` 部件；支持内置 60+ 种表格样式（TableStyleLight/Medium/Dark）；带状行/列自动显示；数据追加时表格范围自动扩展

### 2.6 富文本与高级样式（M20）

- **单元格富文本**：单元格内多段不同格式混排（字体/颜色/大小/粗斜体），通过 `RichTextRun` 列表和 `<r><rPr>` XML 结构实现
- **渐变填充**：线性/径向渐变背景色（`GradientFill` 类型）
- **图案填充**：`PatternFill`（dot/grid/stripe/darkGray 等内置图案，可设前景/背景色）
- **四边独立边框**：上下左右边框可独立设置样式和颜色（Top/Bottom/Left/Right 各一个 `ExcelCellBorderStyle`）
- **字体增强**：删除线（`Strike`）、上标（`Superscript`）、下标（`Subscript`）

### 2.7 图表集成（M21）

- **图表写入**：`ExcelWriter.AddChart(ExcelChart)` 方法将图表集成到普通工作表中，生成 `xl/charts/chartN.xml` + drawing 部件；支持柱状/条形/折线/饼/环形/面积/散点图
- **图表读取**：`ExcelReader.ReadCharts()` 方法解析现有工作表中的图表，提取图表类型、标题、分类标签和系列数据，填充 `ExcelSheet.Charts` 集合

### 2.8 布局增强（M23）

- **文本旋转**：单元格文字旋转角度（0-180 度，`textRotation` 属性）
- **缩进**：单元格文字水平缩进级别（`indent` 属性，1 级 = 3 字符宽）
- **缩小以填充**：当文字超出列宽时自动缩小字号（`shrinkToFit` 属性）
- **列分组/大纲**：`SetColumnOutlineLevel(col, level)` 设置列分组级别（折叠/展开），生成 `<cols outlineLevel>`
- **行分组/大纲**：`SetRowOutlineLevel(row, level)` 设置行分组级别，生成 `<row outlineLevel>` 及工作表大纲设置

### 2.9 条件格式增强（M22）

- **图标集（IconSet）**：箭头（3/4/5 级）、旗标（红/黄/绿）、星级、交通灯等内置图标集，生成 `<cfRule type="iconSet">`
- **自定义公式**：基于公式表达式的条件格式（`<cfRule type="expression"><formula>...`），可实现任意复杂条件

### 2.10 企业特性（M24）

- **工作表标签颜色**：`SetSheetTabColor(sheetIndex, color)` 设置工作表标签颜色，生成 `<sheetPr><tabColor>`
- **工作簿级保护**：`ProtectWorkbook(password, lockStructure, lockWindows)` 防止添加/移动/删除工作表，生成 `<workbookProtection>` 含密码哈希
- **计算选项**：写入 `<calcPr calcId="191029" fullCalcOnLoad="1"/>` 确保文件在 Excel 中打开时自动重新计算

## 3. Word 功能需求

### 3.1 高保真读写架构

NewLife.Office Word 模块采用 **L3 级保真架构**（模型 + RawXml 透传），确保 docx 文件读→改→写往返不丢失格式。核心设计：

- **WordDocument 文档模型**：`WordReader.ReadDocument()` 将 docx 解析为完整的 `WordDocument` 对象图，用户程序化修改后通过 `WordWriter.Save(doc)` 写回。模型覆盖 >85% 常用属性。
- **RawXml 透传兜底**：每个 `WordElement` 保留原始 `<w:p>`/`<w:tbl>` 的 OuterXml，Writer 在模型字段为 null 时直接写入 RawXml，确保未建模属性（删除线、高亮、字符间距、段落边框等）不丢失。
- **ZIP 部件完整透传**：除 `word/document.xml` 外的所有 ZIP 部件（`styles.xml`、`numbering.xml`、`settings.xml`、`header*.xml`、`footer*.xml`、`footnotes.xml`、`endnotes.xml`、`theme/`、`fonts/` 等）原样保留，Writer 在透传模式下仅重建 document.xml。
- **往返保真原则**：读入 → 不改动 → 写出 = 字节级等价（除 XML 空白规范化）；读入 → 修改 Elements → 写出 = 视觉级等价（仅改动部分重新生成，其余 RawXml 原样写入）。

### 3.2 模板填充与占位符

- 基于 docx 模板替换 `{{Key}}` 占位符，保留原有样式（`WordTemplate.Fill()`）
- 嵌套 XML 拆分占位符合并（Word 编辑器可能将 `{{Key}}` 拆到多个 `<w:r>` 中，通过段落级合并后替换自动处理）
- 表格区域填充（`FillTable()`，`{{#ListKey}}`...`{{/ListKey}}` 标记循环区域，自动扩展数据行）
- 图片占位符替换
- 对象属性自动映射（`Fill(path, model)` 从对象属性自动生成占位符字典）

### 3.3 文本与排版

#### 3.3.1 Run 级格式

- 粗体（Bold）、斜体（Italic）、下划线（Underline + UnderlineStyle，支持 single/double/wave/thick/dotted/dash 等多种样式）
- 字体名称（FontName）、字号（FontSize，磅）
- 字体颜色（ForeColor，16 进制 RGB）
- 上标（Superscript）、下标（Subscript）
- 删除线（Strikethrough）
- 高亮/文字背景色（RawXml 兜底）
- 字符间距（CharacterSpacing，缇）、字符缩放（CharacterScaling，百分比）

#### 3.3.2 段落级格式

- 标题样式（Heading 1-6，枚举 + `StyleId` 原始标识符保留）
- 段落对齐（左/中/右/两端对齐，`Alignment`）
- 段落缩进（`IndentLeft`/`IndentRight`/`FirstLineIndent`，twips）
- 段前段后间距（`SpaceBefore`/`SpaceAfter`，twips）
- 行距（`LineSpacingPct`，百分值：100=单倍/150=1.5倍/200=双倍）
- 段落背景色（`BackgroundColor`，16 进制 RGB）
- 段落边框（`WordParagraphBorders`，四边独立设置，多种线型可选）
- 制表位（Tab Stop，`WordTabStop`，支持位置/对齐/前导符）
- 分页符（`IsPageBreak`）
- 多 Run 富文本段落（一个段落包含多种格式的 Run）

#### 3.3.3 列表

- 无序列表（项目符号，`IsBullet`）
- 有序列表（编号）
- 多级嵌套列表（规划增强，RawXml 已兜底）
- 自定义项目符号/编号格式（规划增强，RawXml 已兜底）

### 3.4 页面与布局

- 页面尺寸/方向（`WordPageSettings`：A4/A3/Letter/自定义，横向/纵向）
- 页边距（上/下/左/右，twips）
- 页眉（文本/富文本，`WordHeader` 模型，支持 default/first/even 三种类型）
- 页脚（文本/富文本，`WordFooter` 模型，含页码占位符）
- 奇偶页不同页眉页脚（RawXml 兜底）
- 首页不同页眉页脚（RawXml 兜底）
- 多节不同页面设置（`sectPr` RawXml 兜底）
- 分栏（规划中，RawXml 已兜底）
- 行号（RawXml 兜底）
- 页面边框（RawXml 兜底）
- 水印（文字水印 `WatermarkText`；图片水印规划中）

### 3.5 表格

### 3.6 图片与绘图

- 插入图片（PNG/JPEG，`InsertImage()`，支持宽高厘米/EMU 控制）
- 图片提取（`ExtractImages()`，二进制数据 + 扩展名）
- 图片环绕方式（RawXml 兜底，支持嵌入/四周/紧密等）
- SVG 图片（透传）

### 3.7 导航与引用

## 4. PPT 功能需求

### 4.1 高保真读写架构

NewLife.Office PPT 模块采用 **L2 级保真架构**（完整对象模型 + 媒体/母版 ZIP 部件透传），确保 pptx 文件读→改→写往返不丢失格式。核心设计：

- **PptDocument 文档模型**：`PptxReader.ReadDocument()` 将 pptx 解析为完整的 `PptDocument` 对象图，用户程序化修改后通过 `PptxWriter.Save(stream, document)` 写回。模型覆盖幻灯片内所有主要内容类型（文本框/形状/图片/表格/图表/视频/组/动画/批注/切换/背景）。
- **媒体 / 母版透传**：读取 pptx 时，所有媒体文件（图片/视频/字体）、母版 XML、版式 XML 原样保留；`PptxWriter` 在模板加载模式（`LoadMaster()`）下仅重建 slide XML，其余部件完整透传。
- **往返保真原则**：读入 → 不改动 → 写出 = 媒体字节一致、视觉等价（XML 规范化）；读入 → 修改 Slides → 写出 = 仅改动部分重新生成，母版/版式/主题/嵌入字体原样写入。
- **跨文件合并**：`PptxWriter.Merge()` 和 `CopySlideFrom()` 将幻灯片从一个 pptx 合并到另一个，自动处理媒体文件名冲突和关系 ID 重建。

### 4.2 文件读写

- 读取和创建 pptx（Open XML，直接操作 ZIP + XML，零外部依赖）
- 读取 ppt 格式（OLE2/CFB 容器，兼容 PPT 97-2003，纯文本提取）
- 从文件路径或 `Stream` 读写（`PptxReader` / `PptxWriter` 均支持流接口）
- 多媒体透传：图片/视频/字体嵌入文件保持字节一致
- 模板加载：`new PptxWriter(templatePath)` 一行复用现有 pptx 的母版/版式/主题/嵌入字体

### 4.3 Run 级文本格式

与 ShapeCrawler 对标，支持精确到 `<a:r>` 级别的文本格式控制：

#### 4.3.1 Run（片段）级格式
- 字体名称：拉丁（LatinFontName）、东亚/中文（EastAsianFontName）、复杂脚本（ComplexScriptFontName）、符号（SymbolFontName）四类独立设置
- 字号（FontSize，磅）
- 粗体（Bold）、斜体（Italic）、下划线（Underline）
- 字体颜色（ForeColor，16 进制 RGB 或 schemeClr 主题色引用）
- 渐变文字填充（GradFillColors + GradAngle，两色线性渐变文字）
- 上标/下标（规划中，`<a:rPr baseline>`）
- 超链接（HyperlinkUrl，外部 URL 或幻灯片内部跳转）

#### 4.3.2 段落（Paragraph）级格式
- 对齐方式（Alignment：左/中/右/两端/分散对齐）
- 行距（LineSpacingPct，百分比；LineSpacingPts，固定磅值）
- 段前间距（SpaceBeforePt）、段后间距（SpaceAfterPt）

## 5. PDF 功能需求

### 5.0 高保真读写架构（2026-06 新增）

NewLife.Office PDF 模块采用 **L3 级保真架构**（对象级保留 + 内容流重建），确保 PDF 文件读→改→写往返不丢失内容。核心设计：

- **xref 交叉引用表解析**：`PdfXRefTable` 解析 PDF 文件尾部的交叉引用表（传统 xref 和 PDF 1.5+ xref 流），支持增量更新链（`/Prev`），按对象号精确定位每个间接对象。
- **PDF 对象解析器**：`PdfObjectParser` 将原始 PDF 语法解析为强类型中间表示（`PdfDict`/`PdfArray`/`PdfName`/`PdfString`/`PdfRef` 等），支持嵌套结构和间接引用。
- **内容流重建**：`PdfContentStream` 解析 PDF 页面内容流为操作符序列（`PdfOperator`），支持修改文本/图形操作符后重新序列化回 PDF 语法。
- **FlateDecode 解压缩**：`PdfReader.DecompressFlate()` 使用 DeflateStream 解压 zlib 压缩的内容流，支持多重过滤器链（`/ASCIIHexDecode` + `/FlateDecode`）和对象流（ObjStm）。

### 5.1 创建与生成

- 从零创建 PDF 文件（不依赖完整浏览器引擎）
- 文本排版（字体/大小/颜色/粗体/斜体）、中文字体支持（系统 TrueType 字体嵌入，含 CIDFontType2 + Identity-H + CIDToGIDMap + ToUnicode 完整映射链）
- 多页文档、表格（含斑马纹/表头/边框/合并列）、图片插入、页眉页脚
- **AcroForm 表单创建**（文本框/复选框/下拉框/签名字段，含字段树、外观流、页面注释）

### 5.2 读取与提取

- 文本提取（基于 xref + FlateDecode 解压缩，支持 PDF 1.0-1.7）
- 逐页文本提取、元数据读取（含 xref trailer 优先）
- 页面信息（页数、MediaBox、旋转角度）
- 书签/大纲读取（递归 Outline 树，含层级和目标页面）
- 嵌入图片提取（基于 xref 遍历所有 /Subtype /Image 对象，自动解压 FlateDecode）
- 带坐标位置的文本提取（追踪 Tm/Td 文本矩阵）

### 5.3 编辑与操作

- 合并/拆分 PDF、页面旋转/删除、叠加文字/图片（盖章/水印/页码）
- **对象级编辑基础**（PdfContentStream 解析→修改→重建内容流）
- 表单填充（AcroForm）及书签/大纲支持

### 5.4 高级功能（规划中）

- AES-128/256 加密（当前 RC4 已实现）
- 数字签名（PKCS#7 + 可见签名域）
- PDF/A 合规（1B/2B/3B）
- 注释/批注全部类型写入（当前仅超链接 Link 类型已实现）

### 5.5 Fluent PDF 生成（QuestPDF 风格）

- **声明式布局 API**：支持 Component 嵌套组合（Row / Column / Text / Image / Table），可链式调用。
- **自动分页**：内容超出单页高度时自动扩展到新页，无需手动计算页数。
- **组件复用**：可定义可复用的布局模板块，在多个文档/页面共享。

### 5.6 底层操作（PdfSharp 风格）

- **PDF 合并/拆分**：支持多文件合并、按页拆分导出。
- **低层绘图 API**：画线、矩形等基础图元，适合自定义图表/设计稿场景。

## 7. OLE2/BIFF8 跨格式基础层

### 7.1 OLE2/CFB 容器（Compound File Binary）

OLE2（Compound File Binary，CFB）是 Microsoft Office 97-2003 时代所有二进制文档格式（.xls、.doc、.ppt）共用的容器格式。需从零实现 CFB 解析/写入器，作为 xls/doc/ppt 三种格式的共用基础层。

#### 7.1.1 CFB 读取需求

- 解析复合文档头部（512 字节 Header）：版本、扇区大小、FAT/miniFAT 位置
- FAT（文件分配表）遍历：将散布磁盘的扇区链接为流（Stream）
- mini-FAT 处理：< 4096 字节的小流使用 mini-Stream
- 目录项（Directory Entry）遍历：按红黑树结构枚举所有 Storage / Stream
- 按名称访问流（如 "Workbook"、"WordDocument"、"\1CompObj"）
- 多版本兼容（v3 = 512B 扇区, v4 = 4096B 扇区）

#### 7.1.2 CFB 写入需求

- 从零生成最小合法 CFB 文件
- 写入一个或多个 Stream
- 生成正确的 FAT/mini-FAT 和目录树

### 7.2 Excel xls/BIFF8 格式（M17）

详见第 2 章 M17 模块——在 CFB 容器中读写 BIFF8 格式工作簿流（Workbook stream）。

#### 7.2.1 BIFF8 读取

- 解析 BOF/EOF 记录标识工作表边界
- SST（Shared String Table）记录读取
- DIMENSIONS / ROW / LABELSST / NUMBER / FORMULA / BLANK 核心记录
- XF / FONT / FORMAT 样式记录，还原单元格字体/颜色/数字格式
- NAME 记录（定义名称）、MERGEDCELLS（合并区域）、HYPERLINK（超链接）

#### 7.2.2 BIFF8 写入

- 生成 BOF / SST / DIMENSIONS / ROW / 单元格记录序列
- 支持字符串/数值/日期/布尔类型写入
- 支持基本样式（字体/边框/颜色）

### 7.3 Word doc/MS-DOC 格式（W07）

- 从 CFB 容器中提取 `WordDocument` 和 `1Table` / `0Table` 流
- 解析 FIB（File Information Block）获取文档结构偏移量
- 遍历文本字符（cp）范围，提取段落纯文本
- 识别基本段落格式（标题级别、粗体/斜体）
- 写入支持（低优先级）：生成最小合法 .doc 文件

### 7.4 PPT ppt/MS-PPT 格式（S08）

- 从 CFB 容器中提取 `PowerPoint Document` 流  
- 遍历「User Edit Atom」→「Persist Directory」定位最新版本
- 解析 `SlideAtom` + `TextBytesAtom`/`TextCharsAtom` 提取幻灯片文本
- 写入支持（低优先级）

## 10. ODS（OpenDocument Spreadsheet）

ODS 是 LibreOffice / OpenOffice 的原生电子表格格式（ZIP + OpenDocument XML）。

### 10.1 读取

- 解析 `content.xml` 中的工作表数据（行/列/单元格）
- 单元格类型识别（string/float/date/boolean/formula）
- 读取合并单元格（`table:number-columns-spanned` / `table:number-rows-spanned`）
- 读取基础样式信息（字体/颜色/对齐）

### 10.2 写入

- 生成合法 .ods 文件（`mimetype` + `content.xml` + `styles.xml` + ZIP 结构）
- 多工作表写入
- 基础单元格样式写入（字体粗体/斜体/大小/颜色、背景色、水平对齐）通过 `OdsCellStyle` + `OdsSheet.CellStyles` 注入，自动生成 `office:automatic-styles` 块
- 公式写入（OpenFormula 格式）

### 10.3 与 Excel 互转

- `.ods` → Excel 对象模型（`IEnumerable<Object[]>`）/ DataTable
- `.xlsx` → `.ods` 转换（限基础数据类型和样式）

## 11. 邮件存档（Email）

### 11.1 EML 格式（RFC 5322 + MIME）

- 解析 MIME 头部（From/To/Cc/Bcc/Subject/Date/Message-ID/Reply-To）
- 多部分解析（multipart/mixed/alternative/related）
- 正文提取（text/plain 和 text/html）
- 附件提取（文件名、内容类型、二进制数据）
- 内嵌图片（Content-ID 引用）
- Q 编码 / Base64 解码（RFC 2047）
- 创建/发送 .eml（生成合法 MIME 文档）

### 11.2 MSG 格式（Outlook Message，OLE2 容器）

- 从 OLE2 容器提取 MAPI 属性（PR_SUBJECT / PR_BODY / PR_HTML_BODY / 收件人列表）
- 提取附件（MAPI Attachment 对象）
- 读取发件人/收件人元数据
- 写入支持（低优先级）

## 12. iCalendar（.ics）

### 12.1 读取

- 解析 iCalendar 3.0/4.0 格式（RFC 5545）
- 提取 VEVENT 组件（标题/描述/开始/结束时间/位置/重复规则/提醒）
- 提取 VTODO / VJOURNAL / VFREEBUSY 组件
- 时区处理（VTIMEZONE 组件，UTC 偏移）
- 重复规则（RRULE）解析（每日/每周/每月/每年）

### 12.2 写入

- 生成符合 RFC 5545 的 .ics 文件
- 创建 VEVENT（日程事件）、VTODO（待办事项）
- 支持重复规则（RRULE）写入
- 多日历条目导出

## 13. vCard 联系人名片（.vcf）

### 13.1 读取

- 解析 vCard 3.0/4.0 格式（RFC 6350）
- 提取核心属性：FN/N（姓名）、ORG/TITLE（组织职位）、TEL（电话）、EMAIL、ADR（地址）、PHOTO（头像）、URL、NOTE、BDAY（生日）
- 分组（GROUP）属性解析
- Base64 编码的 PHOTO 提取为字节数据

### 13.2 写入

- 生成符合 RFC 6350 的 .vcf 文件（vCard 4.0 优先，可降级到 3.0）
- 写入所有核心属性
- PHOTO 支持嵌入式 Base64 编码和 URL 引用两种模式

## 14. EPUB 电子书

### 14.1 读取

- 解析 EPUB 2/3 结构（ZIP + OPF/NCX/Nav Document）
- 读取书籍元数据（标题/作者/出版商/ISBN/语言/封面）
- 提取目录结构（NCX/Nav，章节层次）
- 提取各章节 HTML 内容（xhtml 文件）
- 提取嵌入图片、CSS 样式表、字体

### 14.2 写入

- 生成合法 EPUB 3 文件（`mimetype` + `META-INF/container.xml` + OPF + XHTML + ZIP）
- 写入元数据和目录
- 添加章节（XHTML 内容）
- 嵌入封面图片和基础 CSS

## 15. XPS 固定版式文档（.xps / .oxps）

### 15.1 读取

- 解析 XPS 文件结构（ZIP + `/Documents/1/FixedDocumentSequence.fdseq`）
- 提取各页面 XAML（FixedPage）
- 文本内容提取（`Glyphs` 元素 → 文本字符串）
- 图片提取（嵌入 PNG/JPEG 资源）
- 提取页面尺寸、页数

### 15.2 写入

- 生成最小合法 XPS/OXPS 文件
- 写入文本（单一字体，无复杂布局）
- 插入图片
- 写入元数据（标题/作者）

## 5. 竞品优劣势分析

### 5.1 EPPlus

**优势**：功能最全面的开源方案，API 设计友好，文档丰富。  
**劣势**：v5 起商业使用需购买许可证；不支持 xls 格式；内存占用较高。

### 5.2 NPOI

**优势**：开源免费，支持 xls/xlsx，功能覆盖广泛，社区活跃。  
**劣势**：包体积大（~10MB），依赖多，API 较底层，学习曲线陡峭，内存占用高。

### 5.3 ClosedXML

**优势**：API 友好、代码可读性好，MIT 许可。  
**劣势**：依赖 Open XML SDK（较重），不支持 net45，不支持 xls。

### 5.4 MiniExcel

**优势**：极致轻量，流式读写，内存占用极低，支持模板填充和对象映射。  
**劣势**：样式支持有限，不支持合并单元格、冻结窗格等高级特性。

### 5.5 Aspose.Cells

**优势**：功能最全，企业级品质，支持几乎所有 Excel 特性。  
**劣势**：**价格高昂**（单开发者License数千美元），闭源，包体积大。

## 6. NewLife.Office 差异化定位

### 6.1 核心竞争力

| 维度 | NewLife.Office 优势 |
|------|-------------------|
| **轻量** | 极小包体积（<100KB），仅依赖 NewLife.Core |
| **CSV 支持** | CSV 读写已由 NewLife.Core 的 `CsvFile` 类覆盖，无需额外依赖 |
| **全框架** | net45 ~ net10 全覆盖，竞品大多放弃 net45 |
| **免费** | MIT 许可，无商业风险（对比 EPPlus/Aspose） |
| **简洁 API** | 几行代码完成导出，无需理解底层 OOXML |
| **智能类型** | 内置长数字保护、身份证保护、百分比识别 |
| **生态** | 与 NewLife.Core/XCode 等深度集成 |

### 6.2 目标赛道

NewLife.Office 不追求成为全功能 Excel 操作库（那是 EPPlus/Aspose 的领域），而是定位为：

> **轻量级、高性能、零门槛的 Excel 导入导出工具库**

覆盖 90% 的业务导出场景，保持极简的 API 和极小的依赖。

### 6.3 竞品对标

- **vs MiniExcel**：功能更丰富（样式、合并、冻结等），同样轻量；CSV 通过 NewLife.Core CsvFile 支持
- **vs EPPlus**：完全免费，更轻量，框架兼容性更好
- **vs NPOI**：更轻量，API 更简洁，学习成本更低
- **vs ClosedXML**：无 Open XML SDK 依赖，支持 net45

---

## 7. 结论

NewLife.Office 通过“轻量 + 实用 + 免费”的定位，在 .NET Excel 操作库中占据独特位置。当前版本已实现样式、合并、冻结、筛选、对象映射、DataTable、模板填充、图片、页面设置等核心能力，并保持零额外依赖与极小体积优势。结合当前测试结果（63/63 通过），已可覆盖绝大多数业务导入导出场景。
